# ==연산자 와 ===연산자

==연산자(Equal Operator) 와 ===연산자(Strict Equal Operator)는 비슷하게 생겼지만 차이가 있다.

==연산자의 경우 우리가 자주 써왔듯이

```javascript
const a = 1;
const b = "1";
console.log(a == b); //true
```

이렇게 두 변수의 값을 비교하는 연산자이다.

위 코드를 보면 a는 int 형, b는 String 임에도 값이 서로 1로 동일하기에
console.log를 해보면 true라는 값이 출력된다.

그러나 ===연산자는 다르다.
=== 연산자 또한 == 연산자와 같이 값을 비교하는 것 맞으나 변수의 타입 또한 비교한다.

그래서 위 코드에서 ===연산자를 사용하면

```javascript
const a = 1;
cosnt b = "1";
console.log(a===b) //false
```

이렇게 서로 다른 자료형이기에 false라는 값이 출력된다.

그럼 배열 과 객체를 비교할 경우에는 어떨까?

```javascript
const a = [1, 2, 3];
const b = [1, 2, 3];
console.log(a == b);
console.log(a === b);
```

위 코드는 어떤 결과가 나올까?

둘 다 false가 출력된다.

왜냐하면

배열은 메모리에서 자리를 차지하고 a,b는 그 배열의 주소를 참조하는 참조변수이다.

그래서 서로 다른 주소를 가지기 때문에 값 과 데이터 타입이 동일하더라도 false라는 값이 출력되는 것이다.
