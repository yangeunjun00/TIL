# SSG ISR

## SSG

SSG(Static Site Generation)는 렌더링 전략 중 하나이다.

### 작동 방식

SSG는 프로젝트가 빌드되는 시점에서 API에서 데이터를 가져와 정적인 HTML 파일만을 생성해서 렌더링하는 기법이다.

### 장점

서버가 이미 만들어진 파일을 보내주면 되므로 속도가 빠르고 SEO(검색 엔진 최적화)에 유리하다.

### 단점

데이터에 변동이 생기면 다시 빌드를 해야 한다.

## ISR

ISR(Incremental Static Regeneration)은 앞서 말한 SSG의 단점을 해결하기 위해 생긴 렌더링 전략이다.

### 작동 방식

SSG와 동일하게 프로젝트가 빌드되는 시점에 HTML 파일이 생성된다.
설정한 시간이 지난 후 사용자가 웹 사이트에 접속하면 일단은 서버에서 만들어 둔 옛날 페이지를 보여주고

그와 동시에 백그라운드에서는 데이터가 변동 된 새로운 페이지를 생성 후 빌드 시킨다.

그 다음 사용자는 새로 업데이트 된 페이지를 보게 된다.

### 장점

데이터가 변동되도 다시 빌드시키지 않고 특정 페이지만 업데이트 되면 된다.

### 단점

첫 번째 접속자는 옛날 페이지를 보게된다.
