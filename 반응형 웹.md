# 반응형 웹

반응형 웹이란 viewport(브라우저에서 웹 콘텐츠가 사용자에게 보여지는 영역)의 크기에 따라 콘텐츠의 크기나 레이아웃 배치를 적절하게 배치하여 사용자에게 편의성을 주는 UX 기술? 이라고 할 수 있다.

## 사용

### em 과 rem

#### em

em은 부모요소의 크기에 비례한 자식의 크기이다. 아래 예시를 보면

```
<style>
    .bumo{
        font-size: 16px;
    }
    .zasik{
        font-size: 1em;
    }
</style>
<div class="bumo">
    <div class="zasik"></div>
</div>
```

bumo태그의 폰트 크기를 16px로 설정 해두면 zasik 태그의 폰트 또한 16px이 된다.

em 앞의 숫자는 배수이기 때문에 2em이면 32px, 3em이면 48px이 된다.

#### rem

rem은 최상위 요소(root)인 <html>을 기준으로 크기를 설정한다.

```
<style>
    .zasik{
        font-size: 1rem;
    }
</style>
<html>
    <div class="zasik"></div>
</html>
```

이렇게 하면 zasik태그의 폰트 크기는 16px(html의 기본 크기는 16px임)이 된다. rem 또한 em과 동일하게 앞의 숫자가 배수이다.

### %

% 또한 앞서 말한 em 이나 rem 처럼 부모를 기준으로 크기를 설정한다.

그러나 %는 박스나 레이아웃의 크기를 설정하는데 사용한다.

예시

```
<style>
    .bumo{
        width:1000px;
        height:500px;
    }
    .zasik{
        width:50%;
        height:50%;
    }
</style>
<div class="bumo">
    <div class="zasik"></div>
</div>
```

이러면 zisik태그의 크기는 너비 500px에 높이 250px인 박스가 된다.

### vw, vh

vw, vh는 viewport를 기준으로 크기를 설정한다.

```
<style>

    .zasik{
        font-size: 1vw;
    }
</style>
    <div class="zasik"></div>
```

이렇게 하면 viewport의 너비의 1/100(vw 앞의 숫자가 1이기에)만큼의 폰트 크기를 가지게 된다.

vh 또한 viewport의 높이에 비례한다는 걸 알 수 있다.

### media query

media query야 말로 말로만 듣던 그 반응형 웹을 구현하는 문법이다.

```
@media(조건){
    스타일
}
```

이렇게 조건에 해당하게 되면 중괄호 안의 스타일을 적용시키는 문법이다.

```
@media(min-width:300px){
    .winner{
        font-size:8px;
    }
}

<div class="winner">적자생존</div>
```

이러면 viewport의 크기가 300이하가 되었을 때 winner태그의 폰트 크기는 8px이 된다.
