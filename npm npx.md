# npm 과 npx의 차이

우리가 보통 개발을 할 때 여러 라이브러리 혹은 프레임워크를 사용하는데(ex : axios, react-router-dom,Next.js)
이를 우리 프로젝트에 설치할 때 패키지 매니저를 통해 패키지를 설치한다.

```
npm install 패키지명
```

이런 식으로

라이브러리나 프레임워크등 계속 재사용하는 패키지를 설치할 때는 npm 명령어를 통해
해당 프로젝트 폴더의 node_modules폴더에 영구적으로 저장을 시킨다.(이는 프로젝트마다 패키지의 버전을 다르게 할 수 있다는 말이 된다.)

근데 npx는 패키지를 설치하고 관리하는 npm과 다르게 그냥 패키지를 일회용으로 실행만 시키는 명령어이다.

우리가 프로젝트 파일을 생성시킬 때 보통 create-프레임워크명-app 혹은 vite와 같은 프로젝트 생성 도구를 사용해서 생성하는데

이때 npx를 통해 프로젝트 파일 생성 도구를 일회용으로 가져와서 프로젝트 생성을 하는데에 한 번 쓰는 식인거다.

근데 굳이 npx로 일회용으로 프로젝트 파일 생성 도구를 불러와서 생성해야하나?

그냥 npm 명령어로 node_modules 폴더에 프로젝트 파일 생성 도구를 설치해놓아서

원할 때 언제든 사용하면 되는거 아닌가?

라고 생각할 수 있으나

만약 1년전에 그렇게 node_modules파일에 프로젝트 생성 도구를 설치 해놓았다면

1년후에도 1년전 버전의 프로젝트 생성 도구가 실행된다.

그래서 항상

```
npm update -g
```

이 명령어를 통해 버전을 업데이트 해줘야 하는 귀찮음이 있다.

그러나 npx는 항상 **최신 버전**의 패키지를 불러오기 때문에 이런 귀찮은 명령어를 치지 않아도 된다는 장점이 있고 또한 패키지를 설치 하지않고 공용 저장소(npm registory)에서 가져오기 때문에 메모리 용량 측면에서도 이점이 있다.

### 한 줄 설명

npm은 패키지 설치해서 원할 때 마다 패키지 꺼내서 씀(그래서 빠름 근데 메모리 차지함 그리고 패키지 버전 항상 업데이트 해줘야 됨)

npx는 공용 저장소에서 최신 버전 도구 한 번 꺼내서 필요할 때 쓰고 다시 돌려놓음(메모리 차지 안함 설치를 안해서 항상 최신버전임)
