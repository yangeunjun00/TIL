## 디자인 패턴이란?

> 소프트웨어 설계에서 생기는 문제들을 일반적으로 재사용 가능한 **해결책**이다.

디자인 패턴은 코드를 구조화 시켜 개발효율성 과 유지보수성을 높혀준다.

앞으로 설명할 **MVC**가 바로 이 디자인 패턴 중 하나이다.

# MVC

**MVC**는 **M**odel, **V**iew, **C**ontroller의 약자이다.
각각 하는 역할은 다음과 같다.

## 1. 구조

- **Model** (애플리케이션에서 보여지는 실제 데이터, 데이터 처리 및 API 호출)
- **View** (사용자에게 보여지는 UI)
- **Controller** (사용자의 action을 받고 처리하는 부분)

## 2. 동작과정

> - 사용자의 조작(action)을 Controller에서 받아들임

- Controller에서 저장 된 조작에 맞게 Model 업데이트
- Controller가 Model에서 받은 데이터를 사용자에게 보여줄 View를 선택
- View에서 Model에서 받아온 데이터를 렌더링

## 3. 특징

Controller가 보여질 View를 여러개 선택할 수 있는 1:n 구조이다.
View 와 Model 사이에서 높은 의존성이 존재한다.
높은 의존성은 프로젝트의 규모가 커질수록 유지보수성을 저하시킨다.
